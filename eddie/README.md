# eddie — RBY1 로봇 작업 제어 시스템

RBY1 로봇의 **직접교시(마스터암) → 저장 → 자동 재생** 워크플로우를 CLI 메뉴로 제어하는 시스템.

---

## 메뉴 구성

```
--- eddie main ---
  1: RPC 연결
  2: 현재 위치 가져오기
  3: 마스터암 연결
  4: 마스터암 해제
  5: 현재 위치 저장
  6: 저장된 위치 확인
  7: 모바일 베이스 이동
  8: 저장 위치로 이동
  9: 그리퍼 연결/호밍
  0: 그리퍼 수동 제어
  10: 시나리오 자동 실행
  q: 종료
```

---

## 시나리오

### 시나리오 1 — 드릴프레스 작업

모바일 이동 → 왼손으로 물건 픽업 → 드릴프레스로 이동 → 배치 → 오른손 드릴 작업.

| 스텝 | 슬롯 키 | 파일명 | 설명 | 타입 |
|------|---------|--------|------|------|
| 1 | 1-1 | s1_move_start_to_pickup | 시작위치 → 픽업테이블 | 모바일 |
| 2 | 1-2 | s1_left_pickup_approach | 왼팔: 픽업 접근 | 단일위치 |
| 3 | 1-3 | s1_left_pickup_grasp | 왼팔: 물건 파지 | 단일위치 + 왼그리퍼 닫힘 |
| 4 | 1-4 | s1_move_pickup_to_drill | 픽업테이블 → 드릴프레스 | 모바일 |
| 5 | 1-5 | s1_left_drill_place | 왼팔: 드릴프레스 배치 | 단일위치 + 왼그리퍼 열림 |
| 6 | 1-6 | s1_right_drill_approach | 오른팔: 드릴 접근 | 단일위치 |
| 7 | 1-7 | s1_right_drill_grasp | 오른팔: 드릴 파지 | 단일위치 |
| 8 | 1-8 | s1_right_drill_pull | 오른팔: 드릴 당김 | 궤적녹화 |
| 9 | 1-9 | s1_right_drill_push | 오른팔: 드릴 밀기 | 자동생성 (1-8 역순) |

### 시나리오 2 — 경신박스픽업

모바일 이동 → 양손으로 박스 접근 → 파지 → 들기 → 내려놓기 → 놓기.

| 스텝 | 슬롯 키 | 파일명 | 설명 | 타입 |
|------|---------|--------|------|------|
| 1 | 2-1 | s2_move_to_box | 박스 위치로 이동 | 모바일 |
| 2 | 2-2 | s2_box_approach | 양팔: 박스 접근 | 단일위치 |
| 3 | 2-3 | s2_box_grasp | 양팔: 박스 파지 | 단일위치 + 양그리퍼 닫힘 |
| 4 | 2-4 | s2_box_lift | 양팔: 박스 들기 | 단일위치 |
| 5 | 2-5 | s2_box_place | 양팔: 박스 내려놓기 | 단일위치 |
| 6 | 2-6 | s2_box_release | 양팔: 박스 놓기 | 단일위치 + 양그리퍼 열림 |

## 사용법

### 1. 초기 연결

```
1  →  RPC 연결 (192.168.30.1:50051)
9  →  그리퍼 연결/호밍 (Dynamixel 초기화 + 캘리브레이션)
3  →  마스터암 연결 (직접교시 모드 진입)
```

### 2. 위치 티칭 (저장)

```
5  →  시나리오 선택 (1 또는 2)
      →  스텝 번호 선택
      →  마스터암으로 원하는 위치로 이동 후 저장
      →  계속 다른 스텝 저장 가능 (q로 나가기)
```

**저장 타입별 동작:**

| 타입 | 동작 |
|------|------|
| 단일위치 | 현재 관절 위치를 즉시 스냅샷 (전체 26 DOF) |
| 궤적 | 엔터로 녹화 시작 → 마스터암 조작 → 엔터로 종료 (10Hz 샘플링) |
| 자동생성 | 궤적 슬롯의 역순을 자동 생성 (별도 조작 불필요) |
| 모바일 | 이동 시퀀스 입력 (방향 + 시간 + 속도) |

### 3. 모바일 이동 입력 방법

메뉴 5번에서 모바일 슬롯 선택 시, 또는 메뉴 7번에서:

```
방향: f(전진) b(후진) l(좌) r(우) cw(시계회전) ccw(반시계회전)
입력: 방향 시간(초) [속도(m/s 또는 rad/s)]

예시:
  f 3.0         →  전진 3초 (기본 0.2m/s)
  cw 1.05 1.5   →  시계회전 1.05초 (1.5rad/s ≈ 90°)
  b 2.0 0.1     →  후진 2초 (0.1m/s)
  d             →  입력 완료
```

**회전 각도 계산:** `각속도(rad/s) × 시간(초) = 회전량(rad)`
- 90° ≈ 1.571 rad → 기본속도 0.3rad/s 기준 약 5.24초
- 빠르게 하려면: `cw 1.05 1.5` (1.5rad/s × 1.05초 ≈ 90°)

### 4. 개별 위치 이동 (테스트용)

```
8  →  저장된 슬롯 목록에서 선택 → 해당 위치로 이동
      계속 다른 위치 선택 가능 (q로 나가기)
```

### 5. 시나리오 자동 실행

```
10  →  시나리오 선택 (1: 드릴프레스, 2: 경신박스픽업)
       →  저장 안 된 슬롯 경고 → y로 계속 / n으로 취소
       →  모든 스텝 자동 순차 실행
```

## 파일 구조

```
eddie/
├── main.py              # CLI 메뉴 엔트리포인트
├── save_position.py     # 로봇 연결, 시나리오/슬롯 정의, 위치 저장
├── move_position.py     # 위치 이동, 궤적 재생, 시나리오 실행
├── mobile.py            # 모바일 베이스 이동 (SE2Velocity 명령)
├── master_arm.py        # 마스터암 텔레오퍼레이션
├── gripper.py           # Dynamixel 그리퍼 제어
├── endpoint_control.py  # Cartesian 엔드포인트 제어 (독립 기능)
├── connect.py           # 연결 테스트 스크립트
├── positions/           # 저장된 .npy 파일들
│   ├── s1_*.npy         # 시나리오 1 데이터
│   └── s2_*.npy         # 시나리오 2 데이터
└── README.md            # 이 문서
```

## 저장 데이터 형식

| 타입 | numpy shape | 내용 |
|------|-------------|------|
| 단일위치 | `(N,)` | 전체 관절 위치 (rad), N = 로봇 DOF |
| 궤적 | `(F, N)` | F프레임 × N관절, 10Hz 샘플링 |
| 모바일 | `(S, 4)` | S스텝 × [vx, vy, omega, duration] |

## 시나리오 추가 방법

`save_position.py`에서:

1. `SCENARIOS` dict에 새 시나리오 추가
2. `POSITION_SLOTS`에 `{시나리오번호}-{스텝번호}` 키로 슬롯 추가
3. 필요 시 `MOBILE_SLOTS`, `TRAJECTORY_SLOTS`, `AUTO_REVERSE_MAP`, `GRIPPER_ACTIONS`에 키 등록
